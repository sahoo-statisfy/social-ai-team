## Instructions for Claude

When this file is attached, treat it as the binding brand specification for all Statisfy social content. Specifically:

1. Apply the **voice rules** and **banned-phrase list** to every draft — block any draft that contains a banned phrase and rewrite.
2. Use **required terminology** as written (e.g. "AI agents," never "AI copilots" or "AI assistants").
3. Default platform output to **LinkedIn** unless asked otherwise. X is secondary. Instagram / Threads / Facebook / TikTok are off by default — only produce content for them on explicit request for a specific use case (event, recruiting, cross-post).
4. Lead every post with the **outcome and the number** where one is available, then the mechanism.
5. **Never name a customer in a post without confirming permission first.** If the user hasn't confirmed, write the post with a placeholder like `[Customer name — pending permission]` and flag it.
6. Apply the **visual identity tokens** (palette + fonts) when producing image specs, prompts, or layout guidance.
7. Match the **sample patterns** at the bottom of this file for hook + structure + close.

If any user instruction conflicts with this file, surface the conflict and ask before proceeding.

## Voice Rules

**Three-word voice:** Direct. Outcome-focused. Confident.

- **Lead with the outcome, then the mechanism.** "GRR up 2%+" *before* "explainable health scores."
- **Specific over general.** Numbers, named products, named customers (with permission). Never "drives results" — always "drives [specific result, with number]."
- **Action verbs:** saved, delivered, unlocked, generated, automated. Not "helps with," "enables," "supports."
- **Confident, not hyped.** The product is strong enough that copy doesn't need to oversell.
- **Talk to the CS leader.** This person knows the work. Don't define QBR, GRR, NRR, CSM, NB-action — trust the reader.
- **First-person on LinkedIn for thought leadership.** Team members posting in their own voice outperform corporate "we." Brand voice (third-person company) for product news and customer announcements.

### Banned phrases — block on sight and rewrite

- "Revolutionary," "game-changer," "transformative," "paradigm shift," "next-gen"
- "Unlock the power of," "supercharge," "10x"
- "In today's fast-paced world," "Let's dive in," "It's no secret that"
- "I'm excited to share," "At the end of the day," "This is a reminder that"
- "Customer-centric" (everyone says this — say what Statisfy actually does)
- Generic "AI is changing everything" openers

### Required terminology

| Use | Not |
|---|---|
| **AI agents** | AI copilots, AI assistants, AI tools |
| **Stella AI / Predict / Generate / Automate / Statisfy NoteTaker** (Title Case) | stella, predict, etc. (lowercase) |
| **Next-best-action** (hyphenated) | next best action |
| **Health scores** with **explainable AI** | health scoring (when referring to Predict) |
| **Go Live** | "deploy," "launch," "stand up" (when describing time-to-value) |

- *Demo bookings push* → +5% Customer Outcomes, +5% Product, –10% Industry Commentary
- *Authority push* → +10% The Modern CS Org, –5% Product, –5% Industry Commentary
- *Product launch* → +15% Product, –5% from three other pillars
- *Conference / event activation* → +10% Industry Commentary, +5% Customer Outcomes, –10% Frameworks, –5% Product

- **Body:** **Inter** (sans-serif fallback)

### Visual rules

- **One idea per image.** Don't stack stats + framework + quote.

- **Customer logos** require current permission (see roster gate below).

## Calls to Action

In priority order:

1. **Book a Demo** — primary for high-intent and product posts
2. **See the Agents** — discovery-stage, lower-commitment alternative
3. **Watch the [QBR / health score / agent] demo** — product-focused posts
4. **Read the case study** — Customer Outcomes posts
5. **Reply with your take** — opinion / thought-leadership posts (drives comments)

**Never use:** "Click the link in bio" (LinkedIn / X have no such convention), "DM us for info" (B2B buyers don't DM cold), generic "Learn more."

## Quality Checklist — Before Publishing

Every Statisfy social post must pass:

- [ ] Leads with the outcome / number / contrarian take
- [ ] Specific over general — at least one named product, named customer (with permission), or specific number
- [ ] Uses "AI agents" — not copilots, assistants, or tools
- [ ] Product names capitalised correctly (Stella AI, Predict, Generate, Automate, Statisfy NoteTaker)
- [ ] No banned phrases (revolutionary, game-changer, supercharge, 10x, let's dive in, in today's fast-paced world, I'm excited to share, etc.)
- [ ] No invented customer metrics or compliance claims beyond SOC 2 Type II / GDPR / ISO 27001 / HIPAA
- [ ] No comparisons against named competitors
- [ ] Customer naming has current permission OR is marked `[pending permission]`
- [ ] LinkedIn: 3–5 hashtags. X: zero hashtags.
- [ ] Length within platform norms (LinkedIn 150–300 words for most, up to 400 for thought leadership; X ≤ 280 chars per post)
- [ ] CTA is one of the approved Statisfy CTAs (Book a Demo / See the Agents / Watch the demo / Read the case study / Reply with your take)
- [ ] If visual: Navy `#080b1c` background, white text, `#8d57f7` accent, Poppins 600 headline, Inter body, one idea per image

## What This File Does Not Cover

- **Paid campaign mechanics** (budgets, audiences, bidding) — handled by the demand-gen team
- **Press releases / earned media** — different register, handled by PR
- **Internal comms / recruiting collateral** — different audience, lighter brand register
- **Roadmap claims** — only ship messages that match what the product does *today*, not what's planned

When in doubt on any of these, ask the marketing lead before publishing.
